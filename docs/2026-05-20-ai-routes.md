# AI Nutrition Routes — 2026-05-20

## What Was Added

Two new API routes backed by Gemini 2.5 Flash Lite for intelligent nutrition analysis:

### `POST /api/ai/text`
Accepts `{ "description": "grilled chicken with rice and salad" }`. Gemini identifies each food component, estimates portion sizes and macros per item, and returns a `NutritionAnalysis` with individual items and summed totals.

### `POST /api/ai/image`
Accepts `multipart/form-data` with an `image` field (JPEG, PNG, WEBP, HEIC — max 10 MB). Gemini Vision analyzes the photo, identifies visible foods, uses visual cues to estimate portions, and returns the same `NutritionAnalysis` shape.

### Supporting additions
- `src/lib/gemini.ts` — shared constants (`GEMINI_MODEL`, `GEMINI_BASE`, `GEMINI_URL`), `parseJsonFromGemini()` helper, `NUTRITION_ANALYSIS_SCHEMA` prompt template, `EMPTY_ANALYSIS` safe default
- `src/types/index.ts` — `NutritionItem` and `NutritionAnalysis` interfaces added

## Technical Decisions

### Model: gemini-2.5-flash-lite
Chosen per user specification. Faster and cheaper than 2.0 Flash for structured JSON extraction tasks; adequate accuracy for nutrition estimation. The existing search fallback still uses `gemini-2.0-flash` (unchanged) to avoid any regression.

### Shared `gemini.ts` lib instead of duplicating inline
Both routes need the same URL construction, JSON parsing, and schema template. Extracting to `src/lib/gemini.ts` avoids copy-paste and gives a single place to change the model or API base URL later.

### Temperature settings
- Text route: `0.1` — description analysis is deterministic; we want consistent output for the same input
- Image route: `0.2` — visual estimation has inherent ambiguity; a slightly higher temperature avoids identical outputs for similar-but-different photos

### Image sent as `inlineData` (base64), not a URL
Gemini Vision supports both URL references and inline base64. Inline was chosen because:
- Users upload local files — they don't have a public URL
- Avoids storing the image anywhere (no temp files, no object storage)
- 10 MB limit is well within Gemini's inline data capacity

### Never return 5xx to the client
Both routes catch all errors and return `EMPTY_ANALYSIS` (zeroed-out totals) with a 200 status. The frontend can detect failure via `items.length === 0` without crashing. Raw Gemini errors are logged server-side only.

### Frontend untouched
These routes are wired up and ready; the UI integration is a separate step so the backend can be tested independently first.

## How to Test

```bash
# Text route
curl -X POST http://localhost:3001/api/ai/text \
  -H "Content-Type: application/json" \
  -d '{"description":"two scrambled eggs with toast and orange juice"}'

# Image route (replace with a real meal photo path)
curl -X POST http://localhost:3001/api/ai/image \
  -F "image=@/path/to/meal.jpg"
```

Expected response shape for both:
```json
{
  "items": [ { "name": "...", "calories": 0, "protein": 0, "carbs": 0, "fat": 0, "servingSize": "...", "quantity": 1 } ],
  "totalCalories": 0,
  "totalProtein": 0,
  "totalCarbs": 0,
  "totalFat": 0,
  "summary": "..."
}
```
